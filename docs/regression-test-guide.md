# 回归测试系统使用指南

## 概述

AIShareTxt 项目采用**自动化回归测试**系统，确保代码改动不影响核心功能。

> **注意**：由于股票数据实时变化，测试只验证**功能正常运行**，不验证数值完全一致。

---

## 工作流程

```
┌─────────────────────────────────────────────────────────┐
│  开发者修改代码                                          │
│       ↓                                                  │
│  运行本地测试: python scripts/regression_test.py        │
│   - 自动检测未提交更改 (显示 -WIP)                       │
│   - 验证两个核心入口正常工作                             │
│   - 检查输出格式正确                                     │
│       ↓                                                  │
│  测试通过 → 提交代码                                     │
│       ↓                                                  │
│  GitHub Actions 自动运行测试                            │
│   - 在 PR 页面显示测试结果                              │
│       ↓                                                  │
│  项目所有者审查                                         │
│   - CI 显示 ✅ 通过 → 可以合并                          │
│   - CI 显示 ❌ 失败 → 需要修复                           │
└─────────────────────────────────────────────────────────┘
```

---

## 本地使用

### 运行测试

```bash
# 在项目根目录运行
python scripts/regression_test.py
```

### 输出示例

**有未提交更改时：**
```
Git 提交: 64884a2-WIP    ← 注意 -WIP 后缀
分支: main
测试股票: 000001 (平安银行)

✅ API 方式:     通过
✅ 命令行方式:   通过
```

**已提交代码后：**
```
Git 提交: abc1234        ← 正常 commit hash
分支: feature-xxx
...
```

## CI 自动测试

### 触发条件

- 提交 Pull Request 时自动运行
- 可在 Actions 页面手动触发

### 测试内容

| 检查项 | 说明 |
|--------|------|
| API 方式 | `analyze_stock("000001")` 是否正常运行 |
| 命令行方式 | `aishare 000001` 是否正常运行 |
| 输出验证 | 检查输出格式正确，无错误信息 |

### PR 页面显示

测试结果会显示在 PR 的 **Checks** 部分：

```
✅ 回归测试通过

| 检查项 | 状态 |
|--------|------|
| API 方式 (analyze_stock) | ✅ 通过 |
| 命令行方式 (aishare) | ✅ 通过 |

- 测试股票: 000001 (平安银行)
- 测试时间: 2026-03-12 ...
```

## 常见问题

### Q: 为什么不保存快照对比？

A: 股票数据实时变化，价格每天都在变，无法保持"完全一致"。测试的重点是验证代码能正常运行，不报错。

### Q: CI 测试失败了怎么办？

A:
1. 点击查看详细日志
2. 检查是哪个测试失败
3. 修复代码后重新提交

### Q: 本地测试和 CI 结果不一致？

A: 可能是数据源更新导致的，重点检查代码是否有错误，而非具体数值差异。
